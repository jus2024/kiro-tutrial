# AI要約API 要件定義書

## 概要

AI要約APIは、AWS上でサーバーレスに稼働するRESTful APIサービスです。ユーザーがメモを作成・管理し、メモの内容についてAI（AWS Bedrock Claude Sonnet 4.6）に質問できる機能を提供します。

### システムの目的

- メモの作成、取得、更新、削除（CRUD操作）
- 個別メモに対するAI質問応答機能
- 全メモを対象としたAI要約生成機能
- スケーラブルで費用対効果の高いサーバーレスアーキテクチャ

### 技術スタック

- **ランタイム**: Python 3.13（AWS Lambda）
- **API**: Amazon API Gateway（REST API）
- **データベース**: Amazon DynamoDB（シングルテーブルデザイン）
- **AI**: AWS Bedrock Claude Sonnet 4.6（Inference Profile経由）
- **監視**: Amazon CloudWatch
- **IaC**: AWS SAM

## 用語定義

| 用語 | 説明 |
|------|------|
| API Gateway | Amazon API Gatewayを使用したRESTful APIエンドポイント |
| Memo Function | メモのCRUD操作を実行するLambda関数（Python 3.13） |
| AI Function | 個別メモへのAI質問処理を実行するLambda関数 |
| AllMemosSummary Function | 全メモのAI要約を生成するLambda関数 |
| Memo Store | DynamoDBを使用したメモの永続化ストレージ |
| AI Service | AWS Bedrock Claude Sonnet 4.6を使用したAIサービス |
| Memo | メモオブジェクト（ID、タイトル、内容、作成日時、更新日時） |

## 機能要件

### 1. メモの作成

**ユーザーストーリー**: ユーザーとして、新しいメモを作成して、後で参照できる情報を保存したい

**受入基準**:
- 有効なメモ作成リクエストを受信した場合、200ms以内に作成されたメモを返す
- タイトルは1〜200文字の範囲で検証する
- 内容は1〜50,000文字の範囲で検証する
- 入力が無効な場合、400エラーと説明的なエラーメッセージを返す
- メモ作成時、一意のID、作成タイムスタンプ、更新タイムスタンプを付与する
- 生成されたIDを含むメモを返す

**APIエンドポイント**: `POST /memos`

### 2. メモの取得

**ユーザーストーリー**: ユーザーとして、IDで特定のメモを取得して、その内容を表示したい

**受入基準**:
- 有効なメモIDが提供された場合、100ms以内にメモを取得する
- すべてのフィールド（ID、タイトル、内容、作成日時、更新日時）を含むメモを返す
- メモIDが存在しない場合、404エラーと説明的なエラーメッセージを返す

**APIエンドポイント**: `GET /memos/{id}`

### 3. メモの一覧取得

**ユーザーストーリー**: ユーザーとして、すべてのメモのリストを取得して、保存された情報を閲覧したい

**受入基準**:
- リクエストを受信した場合、すべてのメモレコードを取得する
- 更新タイムスタンプの降順でソートされたメモのリストを返す
- ページネーションをサポート（デフォルト20件、最大100件）
- ページネーションパラメータが提供された場合、次のページトークンを含むリクエストされたページを返す

**APIエンドポイント**: `GET /memos`

### 4. メモの更新

**ユーザーストーリー**: ユーザーとして、既存のメモを更新して、その内容を変更したい

**受入基準**:
- 有効な更新リクエストを受信した場合、200ms以内にメモを更新する
- 更新されたタイトルと内容は作成時と同じ基準で検証する
- メモ更新時、更新タイムスタンプを現在時刻に更新する
- すべての現在のフィールドを含む更新されたメモを返す
- メモIDが存在しない場合、404エラーを返す

**APIエンドポイント**: `PUT /memos/{id}`

### 5. メモの削除

**ユーザーストーリー**: ユーザーとして、メモを削除して、不要な情報を削除したい

**受入基準**:
- 有効な削除リクエストを受信した場合、200ms以内にメモを削除する
- 削除成功時、204ステータスコードを返す
- メモIDが存在しない場合、404エラーを返す
- メモ削除時、レコードを完全に削除する

**APIエンドポイント**: `DELETE /memos/{id}`

### 6. 個別メモへのAI質問

**ユーザーストーリー**: ユーザーとして、AIを使用してメモの内容について質問し、洞察と回答を得たい

**受入基準**:
- 有効なAI質問（メモIDと質問）を受信した場合、30秒以内にメモ内容を取得して質問を処理する
- 質問は1〜1,000文字の範囲で検証する
- AWS Bedrock Claude Sonnet 4.6を使用して、メモ内容と質問に基づいて回答を生成する
- 回答テキストと処理メタデータを含むAI回答を返す
- 日本語およびその他のUnicode文字をサポートするため、適切なUTF-8エンコーディングで応答を返す
- メモIDが存在しない場合、404エラーを返す
- AIサービスがエラーを返した場合、指数バックオフで最大3回リトライする
- すべてのリトライが失敗した場合、503エラーを返す

**APIエンドポイント**: `POST /memos/{id}/ask`

### 7. 全メモのAI要約生成

**ユーザーストーリー**: ユーザーとして、すべてのメモのAI要約を生成して、全体のテーマと重要な情報を把握したい

**受入基準**:
- 有効な要約リクエストを受信した場合、すべてのメモレコードを取得する
- すべての取得されたメモの内容を集約する
- AWS Bedrock Claude Sonnet 4.6を使用して、60秒以内に包括的な要約を生成する
- 要約テキスト、処理されたメモ数、処理メタデータを含む要約応答を返す
- 日本語およびその他のUnicode文字をサポートするため、適切なUTF-8エンコーディングで応答を返す
- 総コンテンツサイズがコンテンツ制限を超える場合、最近更新されたメモを優先する
- 制限内でできるだけ多くの完全なメモを含める
- 要約に含まれたメモ数とシステム内の総メモ数を示す
- メモレコードが存在しない場合、情報メッセージを含む200応答を返す
- Bedrockがリトライ可能なエラーを返した場合、指数バックオフで最大3回リトライする
- すべてのリトライが失敗した場合、503エラーを返す

**APIエンドポイント**: `POST /memos/summary`

## 非機能要件

### パフォーマンス

- メモCRUD操作: 200ms以内（p95）
- メモ取得: 100ms以内（p95）
- AI質問処理: 30秒以内（p95）
- 全メモ要約生成: 60秒以内（p95）

### スケーラビリティ

- 自動スケーリングによる可変ワークロードへの対応
- DynamoDBオンデマンド価格設定による予測不可能なワークロードへの対応

### 可用性

- AWS Bedrockインフェレンスプロファイルによる複数リージョンでの自動負荷分散
- リトライロジックによる一時的な障害への対応

### セキュリティ

- IAMロールによる最小権限の原則
- CORS設定によるWebクライアントアクセスのサポート
- API Gatewayレベルでのリクエスト検証

### 監視とロギング

- 構造化JSONログ形式
- CloudWatchメトリクスの発行（リクエスト数、エラー数、処理時間）
- aws-lambda-powertoolsによる構造化ロギングとトレーシング
- エラー発生時の完全なエラー詳細とスタックトレースのログ記録

### データ永続性

- DynamoDBによる永続化ストレージ
- ポイントインタイムリカバリの有効化
- AWS管理キーによる暗号化（SSE）

## API仕様

### エンドポイント一覧

| メソッド | パス | 説明 | タイムアウト |
|---------|------|------|------------|
| POST | /memos | メモの作成 | 10秒 |
| GET | /memos | メモの一覧取得 | 10秒 |
| GET | /memos/{id} | メモの取得 | 10秒 |
| PUT | /memos/{id} | メモの更新 | 10秒 |
| DELETE | /memos/{id} | メモの削除 | 10秒 |
| POST | /memos/{id}/ask | AI質問 | 35秒 |
| POST | /memos/summary | 全メモ要約 | 65秒 |

### 共通レスポンス形式

**成功レスポンス**:
```json
{
  "id": "uuid",
  "title": "string",
  "content": "string",
  "created_at": "ISO 8601 timestamp",
  "updated_at": "ISO 8601 timestamp"
}
```

**エラーレスポンス**:
```json
{
  "error": {
    "code": "string",
    "message": "string",
    "request_id": "string"
  }
}
```

## 制約事項

- タイトル: 1〜200文字
- 内容: 1〜50,000文字
- 質問: 1〜1,000文字
- ページサイズ: 1〜100件（デフォルト20件）
- AI処理の最大トークン数: 180,000トークン
- リトライ回数: 最大3回（指数バックオフ: 1秒、2秒、4秒）
